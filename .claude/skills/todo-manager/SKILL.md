# TODO Manager

スマホファースト × 1セッション1PR のワークフローに最適化した TODO 管理。

## 設計思想

- **スマホが主戦場**: 計画・思考・実行指示すべてスマホから行う
- **1タスク = 1セッション = 1PR**: タスクはPR粒度で切る
- **セルフコンテインド**: 各タスクはそのままClaudeに投げれば実行できるレベルのコンテキストを持つ
- **デスクトップは最低限**: 必要なときだけ使う

## ファイル配置

- **場所**: プロジェクトルート直下の `TODO.md`
- 既存の `TODO.md` があれば読み込んで継続管理する
- なければ新規作成する

## フォーマット

```markdown
# TODO

## now
> 現在のフェーズ・目的を1-2行で。

## next
- [ ] **タスクタイトル** (2026-03-07)
  > やりたいこと、背景、方針をここに書く。
  > Claudeがこのブロックだけ読めば着手できるレベルで。

## backlog
- [ ] タスクタイトル (2026-03-07)
  > コンテキスト。
- [ ] タスクタイトル (2026-03-07)

## inbox
- アイデアや思いつきをそのまま投げ込む場所
- 整理前のメモ、ワンライナーOK

## done
- [x] タスクタイトル (2026-03-01 -> 2026-03-07) #PR-123
- [x] タスクタイトル (2026-02-20 -> 2026-03-01) #PR-122
```

## セクション定義

| セクション | 役割 |
|-----------|------|
| `now` | 現在のフェーズ・方向性。blockquote 1-2行 |
| `next` | 次のセッションでやるタスク。**1つだけ** |
| `backlog` | 優先度順に並んだ待ちタスク。上が高優先 |
| `inbox` | 未整理のアイデア・メモ。形式自由 |
| `done` | 完了済み。PRリンク付き。新しい順 |

## タスクの書き方

### 基本ルール
- 1タスク = 1PR で完結するサイズに切る
- タイトルは太字 `**...**` で短く
- 追加日を末尾に `(YYYY-MM-DD)`
- 完了時は `(追加日 -> 完了日) #PR-N` に更新

### コンテキストブロック（重要）
- `>` blockquote でタスクの直下に書く
- **このブロックだけ読めばClaudeが着手できる**レベルの情報量にする
- 含めるもの: やりたいこと、背景、変更対象ファイル/箇所のヒント、方針・制約
- 長くなりすぎたらタスクを分割するサイン

### サイズの目安
- 大きすぎ: 「認証機能を実装する」→ 分割
- ちょうどいい: 「ログインページのUIを作る」「JWTトークン検証ミドルウェアを追加する」
- 小さすぎはOK: 1PRで出せるなら問題なし

## 操作フロー

### `/todo` - 確認
1. `TODO.md` を読む
2. 簡潔に報告:
   - now（現在のフェーズ）
   - next（次にやること）
   - backlog 件数
   - inbox に未整理があれば件数

### `/todo add` or タスク追加
1. `TODO.md` を読む
2. ユーザーの入力を元にタスクを作成
3. コンテキストブロックを一緒に整理する
4. `backlog` の適切な位置に追加（優先度順）

### `/todo next` - 次のタスクを設定
1. `TODO.md` を読む
2. `next` の現タスクを `backlog` 先頭に戻す（あれば）
3. 指定タスクまたは `backlog` 先頭を `next` に移動

### `/todo done` - 完了
1. `TODO.md` を読む
2. `next` のタスクを `- [x]` にして `done` 先頭に移動
3. 完了日と PR番号を付与（PRがあれば）
4. `backlog` 先頭を `next` に繰り上げ

### `/todo inbox` - 速攻メモ
1. ユーザーの入力をそのまま `inbox` に追加
2. 整理は後回しでOK

### `/todo tidy` - 整理
1. `inbox` の項目をレビュー
2. ユーザーと対話してタスク化 or 削除
3. コンテキストブロックを整理
4. `backlog` の優先度を見直し

## 注意事項

- `next` は常に **1つだけ**。迷いをなくす
- `backlog` は上が高優先度。順番が意思決定
- `inbox` は気軽に。形式不問、整理は `/todo tidy` で
- タスクが大きすぎると感じたら分割を提案する
- コンテキストブロックが空のタスクを見つけたら補完を促す
