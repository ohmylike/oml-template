# TODO Manager

プロジェクトのTODOをルートの `TODO.md` で一元管理する。

## ファイル配置

- **場所**: プロジェクトルート直下の `TODO.md`
- 既存の `TODO.md` があれば読み込んで継続管理する
- なければ新規作成する

## フォーマット

```markdown
# TODO

## 概要
プロジェクトで達成したいことの簡潔な説明。

## タスク
- [ ] タスクの説明 (2026-03-07)
- [ ] タスクの説明 (2026-03-07)
  - [ ] サブタスク（必要な場合のみ）

## 完了済み
- [x] 完了したタスク (2026-03-01 -> 2026-03-07)
```

日付のフォーマット: `YYYY-MM-DD`
- 追加時: タスク末尾に `(追加日)` を付与する
- 完了時: `(追加日 -> 完了日)` に更新する

## ルール

### タスクの書き方
- 1タスク = 1つの具体的な作業単位にする
- チェックボックスで管理: `- [ ]`（未完了）/ `- [x]`（完了）
- 追加時にタスク末尾へ追加日を `(YYYY-MM-DD)` で付与する
- 完了時に `(追加日 -> 完了日)` に更新する
- サブタスクはネストで表現。最大2階層まで
- 曖昧な表現を避け、完了条件が明確な記述にする

### ステータス管理
- 完了したタスクは `- [x]` にチェックを入れ「完了済み」セクションに移動する
- 「タスク」セクションには未完了のものだけを残す

### 更新タイミング
- **作業開始前**: TODO.md を読み、現状を把握する
- **作業完了後**: 完了したタスクを更新する
- **計画変更時**: 新しいタスクの追加や不要タスクの削除を行う

### 概要セクション
- プロジェクトや現在のフェーズの目的を1-3行で記述する
- 大きな方針変更があれば更新する

## 操作フロー

### 新規作成
1. プロジェクトルートに `TODO.md` がないことを確認する
2. ユーザーと目的・タスクを整理する
3. 上記フォーマットで `TODO.md` を作成する

### 確認・報告
1. `TODO.md` を読む
2. 未完了タスク数と完了タスク数を簡潔に報告する

### 更新
1. `TODO.md` を読む
2. 該当タスクのステータスを更新する
3. 完了タスクを「完了済み」セクションに移動する

### タスク追加
1. `TODO.md` を読む
2. 「タスク」セクションの適切な位置に新しいタスクを追加する
